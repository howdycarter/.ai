# Acceptance Criteria

- [ ] A user can create or import bug reports with source, severity, status, owner, and reproduction notes.
- [ ] The dashboard highlights critical bugs, blocked bugs, stale bugs, and unowned bugs.
- [ ] A user can filter by severity, status, owner, and source without losing context.
- [ ] The app includes empty, loading, error, and no-results states suitable for an internal tool.
- [ ] The app shows a concise triage summary that could be pasted into a team update.
- [ ] The final build explains what integrations and permissions would be required for production use.
