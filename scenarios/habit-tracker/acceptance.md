# Acceptance Criteria

- [ ] A user can create, edit, pause, and archive habits.
- [ ] A user can mark daily completion and see the current streak without punishing missed days.
- [ ] The app shows weekly progress and at least one useful pattern or reflection prompt.
- [ ] The app includes empty, validation, and recovery states for first use and missed days.
- [ ] The interface keeps the main daily check-in fast on mobile and desktop.
- [ ] The final build explains what data should be persisted and what privacy expectations apply.
