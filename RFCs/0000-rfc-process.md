# RFC 0000: `.ai` Standard Change Process

## Status

Accepted.

## Purpose

The `.ai` Standard should move quickly without becoming arbitrary. RFCs are
required for changes that affect interoperability, conformance, schemas,
manifest fields, lifecycle states, or pack compatibility.

## Process

1. Create `RFCs/NNNN-short-title.md`.
2. State the problem, proposal, compatibility impact, and migration notes.
3. Open a pull request and request maintainer review.
4. Leave the RFC open long enough for meaningful feedback.
5. Merge when the standard lead or delegated maintainer accepts it.

## Lightweight Changes

Docs, examples, copy edits, new showcase entries, and non-breaking pack additions
do not require RFCs.

## Compatibility

Accepted RFCs must explain how older `.ai` projects remain readable or how users
will migrate.
